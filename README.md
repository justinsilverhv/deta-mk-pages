# deta-mk-pages

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态 HTML 页面集合的存储与分发点，方便长期维护和访问。

## 目录说明

```
.
├── pages/          # 存放所有归档的 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── index.html      # 可选：页面索引或导航页
└── README.md       # 本文件
```

- `pages/` 目录是主要归档位置，每个 HTML 文件独立存放。
- `assets/` 目录存放页面引用的公共资源，可按需组织子目录。
- 根目录下的 `index.html` 为可选入口，用于统一展示或跳转。

## 页面归档说明

- 所有页面以纯 HTML 文件形式存放，不依赖后端服务或数据库。
- 页面之间相互独立，无跨域或引用依赖问题。
- 文件名建议使用有意义的英文或拼音，避免特殊字符。
- 每个页面可以附带独立的资源文件（如 CSS、JS），建议集中存放于 `assets/` 下以保持整洁。

## 维护说明

- 提交新页面时，请将 HTML 文件放入 `pages/` 目录，并确保引用路径正确。
- 如需修改已有页面，请直接编辑对应文件，并更新相关资源。
- 不建议在仓库内放置敏感信息或隐私内容，因为仓库默认公开。
- 欢迎通过 Pull Request 提交页面更新，或通过 Issues 讨论归档需求。
